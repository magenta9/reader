# Let Local Release Transaction Own macOS Publication and Installation

Every local VoiceReader package, deploy, or standalone installation runs inside one repository-owned Local Release Transaction. The transaction acquires `.local-release/lock` atomically before verification or artifact work, writes owner metadata inside that lock, and keeps its candidate under a uniquely identified `.local-release/transactions/<id>/` workspace. A second transaction fails closed with the exact lock path; it never guesses that an existing lock is stale and never cleans another transaction's candidate, staging, backup, or failed artifact. `make clean` deliberately preserves `.local-release/` so the verification phase cannot erase the lock or candidate of the deployment that invoked it.

Packaging builds and verifies the application and DMG in the transaction workspace. Deploy passes that exact candidate through packaged smoke and installation instead of reopening `release/mac/VoiceReader.app`. Only after candidate smoke succeeds does publication stage a complete copy beside `release/mac`, move the previous release to a transaction-owned backup, and commit the new directory by rename. A failed commit restores the previous release. Installation follows the same ownership rule for `/Applications/VoiceReader.app`: the installer requires an explicit transaction and candidate handoff, and safe replacement receives transaction-owned staging, backup, and failed paths. Successful cleanup, rollback, and deliberately preserved recovery artifacts operate only through that capability; prefix-wide scans are forbidden.

The lock is fail-closed across crashes. If a command reports an existing lock, contributors inspect `.local-release/lock/owner.json`, verify that no package, deploy, or install process for this repository is still running, and remove only `.local-release/lock` before retrying. They do not delete `.local-release/` recursively, do not clean transaction workspaces as a side effect of lock recovery, and do not touch `/Applications/VoiceReader.app` or VoiceReader user data. Orphan workspaces are inert because every future transaction uses a new token-qualified identity; they may be diagnosed separately rather than guessed stale by production code.

The highest behavior seam uses a real temporary repository root: hold one transaction across the repository clean gate, prove a competitor fails without touching active candidate or swap resources, inject publication commit failure and verify the previous release is restored, and inject installed verification/rollback failures while preserving the recoverable application. Real `make package-mac` plus packaged smoke continues to prove Release Identity, signing, DMG, isolated userData and runtime behavior.

This decision refines ADR-0024's verified local macOS release chain and ADR-0035's Release Identity handoff without changing their product, signing, artifact naming, installed destination, running-application refusal, or user-data guarantees. Rollback proceeds contract → migrate → expand: first restore the explicit legacy adapters if required, then restore the pre-transaction orchestration, and finally remove the additive transaction owner. Prefix-wide cleanup must not be restored independently. There is no SQLite, IPC, configuration, product-domain, or user-data migration.
