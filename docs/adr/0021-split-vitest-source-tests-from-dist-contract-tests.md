# Split Vitest Source Tests From Dist Contract Tests

VoiceReader will use Vitest for source-level behavior tests and jsdom React UI tests, while keeping a separate build-output contract test for `dist/`, bundled HTML/CSS, native addon copying, package-script assumptions, and cross-process boundary checks. `pnpm test` should become the fast Vitest command over root-level `tests/`, `pnpm test:dist` should run the build-output contract checks, and `pnpm verify` should run typecheck, Vitest, build, then dist contract checks without rebuilding twice; this keeps normal test feedback fast without losing the existing protection around Electron build artifacts. React UI tests will use Testing Library with jsdom, and renderer/overlay entrypoints should be split so tests import pure components with fake bridges rather than triggering `createRoot` or real `window.voiceReader` access at module import time.
