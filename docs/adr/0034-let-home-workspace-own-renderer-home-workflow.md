# Let Home Workspace Own the Renderer Home Workflow

VoiceReader will place the Reader Window's Home setup aggregation, language and Voice projection, recovery actions, Reading Target start or skip feedback and command coordination behind one route-scoped Home Workspace model. The workspace reads saved Settings and MiniMax credential presence as one preparation snapshot, derives readiness and recovery from that authoritative state, and exposes an immutable presentation snapshot plus semantic intents. The React Home view only renders that snapshot, converts DOM events to intents, and maps an open-Settings recovery result to navigation. Playback Session activity and terminal feedback remain main-owned and continue to route through the session's named Feedback Surface under ADR-0026.

Each visit has its own generation. Leaving Home disposes the visit, clears optimistic and pending presentation, and prevents late setup reads or commands from updating or unlocking a later visit; React StrictMode setup replay is treated as a fresh generation of the same route-scoped workspace. Verification, Voice refresh and the Reading Target start intent are single-flight within their lanes. Preferred Voice uses optimistic presentation with a serialized, coalescing latest-intent lane: the first in-flight write completes, only the newest unsent intent remains, a failure rolls presentation back to authoritative Settings, and feedback ownership prevents one lane's late completion from erasing a newer lane's result.

ADR-0030 remains the cross-process authority: Home Workspace consumes the existing Reader Window role bridge and introduces no channel, payload or permission change. Main-owned commands and SQLite Settings remain authoritative; already accepted commands are not rolled back when a route visit ends. This decision parallels ADR-0033's renderer workflow boundary for Settings but keeps Home and Settings as separate route-scoped workspaces because their resources, transient state and command policies differ.

The highest test seam is the Home Workspace interface with a fake Reader Window capability adapter. Reader Window DOM tests retain only visible wiring, StrictMode replay, language/Voice picker compatibility and route re-entry tracers. This is a source-only ownership change: it does not alter public bridge contracts, SQLite schema, MiniMax behavior, Reading History, Playback Session semantics, configuration or user data, and rollback requires no migration.
